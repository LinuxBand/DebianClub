# AIツール選定比較

当サイトのAIセクションでは数十のツールを紹介しています。このページではそれらを横断的に比較し、ニーズに応じて素早く選定できるようにします。各ツールのインストールと詳細な使い方は対応するリンクをクリックしてください。

> 価格情報は公式ポリシーにより変わる可能性があります。以下の表は参考程度にご利用いただき、各製品の公式サイトをご確認ください。「ローカル」とは、自分のマシンでオフライン実行でき、データがマシン外に出ないことを意味します。

|---|---|---|---|---|

## ローカルLLM実行環境

自分のマシンでオフラインにLLMを実行。**データは完全にマシン外に出ません**。プライバシー重視の方やAPI費用を節約したい方向け。ある程度のメモリ/VRAMが必要。

| ツール | 位置づけ | 習熟難易度 | 備考 |
|---|---|---|---|
| [Ollama](/en/ai/ollama) | 一行コマンドでモデルを取得・実行 | ⭐ 最も簡単 | ローカルモデルのファーストチョイス、エコシステム広い |
| [LM Studio](/en/ai/lm-studio) | GUI | ⭐ 簡単 | クローズドソース無料、コマンドラインが苦手な方向け |
| [Jan](/en/ai/jan) | GUI、オープンソース | ⭐ 簡単 | LM Studioのオープンソース代替 |
| [llama.cpp](/en/ai/llama-cpp) | 低レベル推論エンジン | ⭐⭐⭐ 上級 | 最高性能、多くのツールの基盤 |
| [llamafile](/en/ai/llamafile) | 単一ファイル実行可能モデル | ⭐⭐ 中級 | モデルを1ファイルにパッケージ |
| [LocalAI](/en/ai/localai) | OpenAI API互換のローカルサービス | ⭐⭐ 中級 | 既存アプリにローカルバックエンドを提供 |

## どう選ぶ？シナリオ別

- **エディタでコードを書きたい、とにかく手軽** → [Cursor](/en/ai/cursor) または [Windsurf](/en/ai/windsurf)。
- **コマンドラインを多用、複数ファイルにまたがる大規模変更** → [Claude Code](/en/ai/claude-code)；無料で多く使いたい → [Gemini CLI](/en/ai/gemini-cli)。
- **オープンソースアシスタント + 自前モデル/ローカルモデルを使いたい** → [Aider](/en/ai/aider)、[Cline](/en/ai/cline) または [Continue](/en/ai/continue)。
- **プライバシー重視 / オフライン / API費用節約** → [Ollama](/en/ai/ollama)（コマンドライン）または [LM Studio](/en/ai/lm-studio)、[Jan](/en/ai/jan)（GUI）でローカルモデルを実行。
- **最高の推論性能を追求、低レベルカスタマイズ** → [llama.cpp](/en/ai/llama-cpp)。
- **画像生成** → 入門は [Stable Diffusion WebUI](/en/ai/stable-diffusion)、複雑なワークフローは [ComfyUI](/en/ai/comfyui)。
- **AIアプリ/自動化を構築** → [Dify](/en/ai/dify) または [n8n](/en/ai/n8n)。

## ハードウェアヒント

- **クラウドツール**（Cursor、Claude Code、Copilotなど）はマシンへの要求は低いですが、インターネット接続が必要で、データはサービス事業者に送信されます。
- **ローカルモデル**はメモリ/VRAMが必要：7B規模のモデルであれば約8GBメモリで試せます。より大きなモデルではNVIDIAの専用GPUを推奨（[NVIDIA OptimusデュアルGPU設定](/ja/administration/nvidia-optimus)参照）。
