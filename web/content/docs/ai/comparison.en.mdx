## Local Large Model Runtimes

Run large models offline on your own machine—**data never leaves your device**, suitable for privacy-sensitive users or those wanting to save API costs. Requires some memory/VRAM.

| Tool | Positioning | Ease of Use | Notes |
|---|---|---|---|
| [Ollama](/en/ai/ollama) | Pull and run models with one command | ⭐ Easiest | First choice for local models, wide ecosystem |
| [LM Studio](/en/ai/lm-studio) | Graphical interface | ⭐ Easy | Closed-source free, suitable for non-command-line users |
| [Jan](/en/ai/jan) | Graphical interface, open source | ⭐ Easy | Open-source alternative to LM Studio |

## How to Choose? By Scenario

- **Want to write code in an editor, most hassle-free** → [Cursor](/en/ai/cursor) or [Windsurf](/en/ai/windsurf).
- **Heavy command-line user, need cross-file major changes** → [Claude Code](/en/ai/claude-code); want more free usage → [Gemini CLI](/en/ai/gemini-cli).
- **Want open-source assistant with your own model / local model** → [Aider](/en/ai/aider), [Cline](/en/ai/cline), or [Continue](/en/ai/continue).
- **Privacy-conscious / want offline / save API costs** → Use [Ollama](/en/ai/ollama) (command-line) or [LM Studio](/en/ai/lm-studio)/[Jan](/en/ai/jan) (graphical) to run local models.
- **Pursue extreme inference performance, need low-level customization** → [llama.cpp](/en/ai/llama-cpp).
- **Do text-to-image** → Start with [Stable Diffusion WebUI](/en/ai/stable-diffusion), use [ComfyUI](/en/ai/comfyui) for complex workflows.
- **Build AI applications / automation** → [Dify](/en/ai/dify) or [n8n](/en/ai/n8n).

## Hardware Tips

- **Cloud tools** (Cursor, Claude Code, Copilot, etc.) have low local hardware requirements but need internet, and data is sent to the service provider.
- **Local models** require memory/VRAM: A 7B-level model may work with about 8GB RAM; larger models are recommended with an NVIDIA dedicated GPU (see [NVIDIA Optimus dual graphics configuration](/en/administration/nvidia-optimus)).
