# Comparação de Seleção de Ferramentas de IA

A seção de IA deste site apresenta dezenas de ferramentas. Esta página faz uma comparação horizontal para ajudá-lo a escolher rapidamente de acordo com suas necessidades. Clique nos links correspondentes para instalação e uso detalhado de cada ferramenta.

> As informações de preços podem mudar com as políticas oficiais. A tabela abaixo é apenas para referência; consulte os sites oficiais de cada produto. "Local" significa que pode ser executado offline em sua própria máquina, sem que os dados saiam do dispositivo.

## Editores de IA / IDE

Adequado para desenvolvedores que desejam "integração nativa de IA no editor".

| Ferramenta | Forma | Local/Nuvem | Modelo de Cobrança | Público-Alvo |
|---|---|---|---|---|
| [Cursor](/en/ai/cursor) | Editor independente (derivado do VS Code) | Modelo em nuvem | Grátis + Assinatura | Quem quer um IDE AI pronto para uso |
| [Windsurf](/en/ai/windsurf) | Editor independente | Modelo em nuvem | Grátis + Assinatura | Prefere fluxo de trabalho baseado em agentes |
| [Zed](/en/ai/zed) | Editor nativo de alto desempenho | Modelo em nuvem / pode usar local | Código aberto, recursos de IA gratuitos | Busca leveza e velocidade |

## Assistentes de Codificação de IA (CLI)

Agentes de codificação que funcionam no terminal, adequados para quem está acostumado com a linha de comando e precisa fazer alterações entre arquivos.

| Ferramenta | Modelo de Backend | Local/Nuvem | Modelo de Cobrança | Observações |
|---|---|---|---|---|
| [Claude Code](/en/ai/claude-code) | Anthropic Claude | Nuvem | Assinatura ou cobrança por API | Agente nativo do terminal, bom para grandes alterações |
| [Gemini CLI](/en/ai/gemini-cli) | Google Gemini | Nuvem | Grátis + cobrança por API | Possui limites gratuitos generosos |
| [GitHub Copilot](/en/ai/github-copilot) | Vários modelos | Nuvem | Assinatura (pessoal/empresarial) | Ecossistema maduro, ampla integração com IDE |
| [Aider](/en/ai/aider) | Pode conectar qualquer modelo/local | Nuvem ou local | Código aberto (cobrança conforme modelo usado) | Integração profunda com Git |
| [Cline](/en/ai/cline) | Pode conectar qualquer modelo/local | Nuvem ou local | Código aberto (cobrança conforme modelo usado) | Forma de plugin do VS Code |
| [Continue](/en/ai/continue) | Pode conectar qualquer modelo/local | Nuvem ou local | Código aberto (cobrança conforme modelo usado) | Altamente personalizável |
| [OpenCode](/en/ai/opencode) | Pode conectar qualquer modelo/local | Nuvem ou local | Código aberto | Agente de codificação no terminal |

## Tempo de Execução de Modelos Grandes Locais

Execute modelos grandes offline em sua própria máquina, **os dados nunca saem do dispositivo**, adequado para usuários sensíveis à privacidade ou que desejam economizar custos de API. Requer uma certa quantidade de memória/VRAM.

| Ferramenta | Posicionamento | Dificuldade de uso | Observações |
|---|---|---|---|
| [Ollama](/en/ai/ollama) | Baixe e execute modelos com um comando | ⭐ Mais fácil | Primeira escolha para modelos locais, ecossistema amplo |
| [LM Studio](/en/ai/lm-studio) | Interface gráfica | ⭐ Fácil | Gratuito fechado, adequado para usuários que não usam linha de comando |
| [Jan](/en/ai/jan) | Interface gráfica, código aberto | ⭐ Fácil | Alternativa de código aberto ao LM Studio |
| [llama.cpp](/en/ai/llama-cpp) | Motor de inferência de baixo nível | ⭐⭐⭐ Avançado | Desempenho extremo, base para muitas ferramentas |
| [llamafile](/en/ai/llamafile) | Modelo executável em arquivo único | ⭐⭐ Médio | Empacota o modelo em um único arquivo |
| [LocalAI](/en/ai/localai) | Serviço local compatível com a API OpenAI | ⭐⭐ Médio | Backend local para aplicações existentes |

## Como escolher? Por cenário

- **Quer codificar no editor, o mais descomplicado** → [Cursor](/en/ai/cursor) ou [Windsurf](/en/ai/windsurf).
- **Usuário pesado de linha de comando, precisa fazer grandes alterações entre arquivos** → [Claude Code](/en/ai/claude-code); para usar mais de graça → [Gemini CLI](/en/ai/gemini-cli).
- **Quer assistente de código aberto + seu próprio modelo/modelo local** → [Aider](/en/ai/aider), [Cline](/en/ai/cline) ou [Continue](/en/ai/continue).
- **Privacidade / quer offline / economizar custos de API** → Use [Ollama](/en/ai/ollama) (linha de comando) ou [LM Studio](/en/ai/lm-studio), [Jan](/en/ai/jan) (interface gráfica) para executar modelos locais.
- **Busca desempenho de inferência extremo, personalização de baixo nível** → [llama.cpp](/en/ai/llama-cpp).
- **Para gerar imagens a partir de texto** → Iniciantes usem [Stable Diffusion WebUI](/en/ai/stable-diffusion), para fluxos complexos use [ComfyUI](/en/ai/comfyui).
- **Criar aplicações / automação de IA** → [Dify](/en/ai/dify) ou [n8n](/en/ai/n8n).

## Dicas de Hardware

- **Ferramentas na nuvem** (Cursor, Claude Code, Copilot, etc.) têm baixos requisitos de configuração local, mas precisam de internet e os dados são enviados ao provedor.
- **Modelos locais** têm requisitos de memória/VRAM: modelos da escala de 7B precisam de cerca de 8GB de RAM para tentar; modelos maiores recomendam placa de vídeo NVIDIA dedicada (consulte [Configuração de placa dupla NVIDIA Optimus](/pt/administration/nvidia-optimus)).
