# 시스템 요구사항 확인

Debian 13을 설치하기 전에 컴퓨터가 기본 요구사항을 충족하는지 확인하는 것이 매우 중요합니다. 이 페이지는 하드웨어 호환성과 시스템 요구사항을 확인하는 데 도움을 드립니다.

## 🖥️ 하드웨어 요구사항

### 최소 시스템 요구사항

| 구성 요소 | 최소 요구사항 | 설명 |
|------|----------|------|
| **프로세서** | 1 GHz 싱글 코어 | amd64 아키텍처(권장) 또는 arm64 |
| **메모리** | 2 GB RAM | 데스크탑 환경은 더 많은 메모리 필요 |
| **저장 공간** | 25 GB | 기본 시스템 및 응용 프로그램 포함 |
| **그래픽 카드** | 1024×768 지원 | 통합 그래픽으로 충분 |
| **네트워크** | 유선 또는 무선 | 설치 후 소프트웨어 저장소 구성 |

### 권장 구성

| 구성 요소 | 권장 구성 | 장점 |
|------|----------|------|
| **프로세서** | 듀얼 코어 2 GHz+ | 더 원활한 멀티태스킹 |
| **메모리** | 8 GB RAM+ | 여러 응용 프로그램 동시 실행 지원 |
| **저장 공간** | 50 GB SSD | 더 빠른 부팅 및 응답 속도 |
| **그래픽 카드** | 독립 그래픽 카드 | 하드웨어 가속 및 게임 지원 |

::: tip 💡 초보자 팁
하드웨어 구성이 확실하지 않다면 Windows에서 <span class="kbd">Win + R</span>을 누르고 `dxdiag`를 입력하여 시스템 정보를 확인하세요.
:::

## 🔍 하드웨어 호환성 확인

### 1. 프로세서 아키텍처 확인

Debian 13은 다양한 프로세서 아키텍처를 지원합니다:

#### 지원되는 아키텍처
- **amd64** (64비트) - Intel/AMD 64비트 프로세서(주류 권장)
- **arm64** - ARM 64비트 프로세서(새 Mac, 라즈베리 파이 4 등)
- **armhf** - ARM 32비트 프로세서(구형 라즈베리 파이 등)
- **riscv64** - RISC-V 64비트 프로세서(🆕 Debian 13 신규 추가)
- **ppc64el** - IBM Power 리틀 엔디안 64비트 프로세서
- **s390x** - IBM z/Architecture 64비트 프로세서

::: warning i386(32비트) 지원 상태
Debian 13(Trixie)은 **더 이상 i386을 일반 설치 아키텍처로 제공하지 않습니다**. 공식 i386 설치 커널이 없으며, i386용 Debian Installer도 없습니다. Debian에서 i386이 유지되는 용도는: amd64 시스템에서 멀티아키텍처(multiarch) 모드로 구형 32비트 응용 프로그램을 실행하기 위한 것이며, 독립적인 i386 시스템 전체를 설치하기 위한 것이 아닙니다. 하드웨어가 실제로 32비트만 지원한다면, 여전히 i386 설치판을 유지하는 antiX Linux 등의 배포판을 참고하세요.
:::

#### 아키텍처 확인 방법

**Windows에서:**
```cmd
# Win + R을 누르고 cmd를 입력한 후 실행:
wmic computersystem get systemtype
```

**기존 Linux에서:**
```bash
# 프로세서 아키텍처 확인
uname -m
```

### 2. 메모리 확인

#### 다양한 데스크탑 환경의 메모리 요구사항

| 데스크탑 환경 | 최소 메모리 | 권장 메모리 | 특징 |
|----------|----------|----------|------|
| **GNOME** | 4 GB | 8 GB | 현대적, 기능 풍부 |
| **KDE Plasma** | 4 GB | 8 GB | 높은 수준의 사용자 정의 가능 |
| **Xfce** | 2 GB | 4 GB | 경량, 구형 시스템에 적합 |
| **LXDE** | 1 GB | 2 GB | 극도로 간결하고 가벼움 |
| **명령줄** | 512 MB | 1 GB | 텍스트 인터페이스만 |

::: warning ⚠️ 메모리 부족의 영향
메모리가 부족하면 시스템이 스왑 공간을 자주 사용하게 되어 성능이 심각하게 저하될 수 있습니다.
:::

### 3. 저장 공간 확인

#### 디스크 공간 할당 권장 사항

```bash
# 권장 파티션 구성(총 50GB 공간)
/          # 루트 파티션: 25 GB
/home      # 사용자 디렉토리: 20 GB  
swap       # 스왑 파티션: 4 GB(메모리 크기와 동일)
/boot/efi  # EFI 파티션: 1 GB(UEFI 시스템)
```

#### 다양한 용도의 공간 요구사항

| 용도 | 최소 공간 | 권장 공간 | 설명 |
|------|----------|----------|------|
| **기본 시스템** | 10 GB | 20 GB | 핵심 시스템만 |
| **데스크탑 사용** | 25 GB | 50 GB | 오피스 소프트웨어 포함 |
| **개발 환경** | 50 GB | 100 GB | 개발 도구 포함 |
| **멀티미디어 작업** | 100 GB | 500 GB+ | 비디오 편집 등 |

## 🔧 하드웨어 호환성

### 그래픽 카드 호환성

#### NVIDIA 그래픽 카드
```bash
# Debian의 NVIDIA 지원
✅ 오픈소스 드라이버: nouveau(기본값)
✅ 공식 드라이버: nvidia-driver(추가 설치 필요)
⚠️  구형 모델은 지원이 제한될 수 있음
```

#### AMD 그래픽 카드
```bash
# AMD 그래픽 카드 지원
✅ 오픈소스 드라이버: radeon/amdgpu(기본값)
✅ 공식 드라이버: amdgpu-pro(선택 사항)
✅ 대부분의 모델 지원 우수
```

#### Intel 통합 그래픽
```bash
# Intel 그래픽 카드 지원
✅ 오픈소스 드라이버: i915(기본값)
✅ 지원 우수, 바로 사용 가능
✅ 하드웨어 가속 지원 우수
```

### 네트워크 장치 호환성

#### Wi-Fi 칩셋
대부분의 현대 Wi-Fi 칩셋은 우수한 지원을 받지만, 일부는 비자유 펌웨어가 필요합니다:

| 제조사 | 지원 상태 | 비고 |
|------|----------|------|
| **Intel** | ✅ 우수 | 펌웨어 패키지 필요 |
| **Broadcom** | ⚠️ 보통 | 일부 모델은 독점 드라이버 필요 |
| **Realtek** | ✅ 양호 | 대부분의 모델 지원 |
| **Atheros** | ✅ 우수 | 오픈소스 드라이버 지원 우수 |

#### 유선 네트워크
거의 모든 유선 네트워크 인터페이스가 완벽하게 지원됩니다.

## 📋 설치 전 확인 체크리스트

### 필수 확인 항목

- [ ] **프로세서**: 아키텍처 확인(amd64/arm64, 현대 PC는 amd64 권장)
- [ ] **메모리**: 최소 2 GB, 권장 8 GB+
- [ ] **저장 공간**: 최소 25 GB 사용 가능 공간
- [ ] **네트워크**: 네트워크 연결 가능 확인
- [ ] **부팅 방식**: UEFI 또는 Legacy BIOS 확인

### 권장 확인 항목

- [ ] **그래픽 카드**: 모델 확인, 해당 드라이버 준비
- [ ] **Wi-Fi**: 칩셋 모델 기록
- [ ] **프린터**: 브랜드 및 모델 확인
- [ ] **블루투스**: 필요 여부 확인
- [ ] **사운드 카드**: 일반적으로 모두 지원

## 🔬 시스템 정보 수집

### 하드웨어 정보 수집

자세한 하드웨어 정보를 원한다면 다음 방법을 사용하세요:

#### Windows에서
1. <span class="kbd">Win + R</span> 누르기
2. `msinfo32` 입력 후 엔터
3. "시스템 요약" 정보 확인

#### Live USB 사용
```bash
# Debian Live USB 부팅 후 실행
sudo lshw -short          # 하드웨어 개요
lscpu                     # CPU 정보
free -h                   # 메모리 정보
lsblk                     # 저장 장치
lspci                     # PCI 장치
lsusb                     # USB 장치
```

## ⚡ 성능 최적화 권장 사항

### SSD vs HDD

| 저장 매체 유형 | 장점 | 단점 | 적용 시나리오 |
|----------|------|------|----------|
| **SSD** | 속도 빠름, 무소음 | 가격이 비쌈 | 시스템 드라이브로 권장 |
| **HDD** | 가격 저렴, 용량 큼 | 속도 느림, 소음 있음 | 파일 저장에 적합 |

### 듀얼 하드 드라이브 구성 권장
```bash
SSD (120GB+) → 시스템 + 프로그램
HDD (1TB+)   → 문서 + 미디어 파일
```

## 🆘 호환성 문제 해결

### 일반적인 문제

#### Wi-Fi 사용 불가
**해결 방법:**
1. 비자유 펌웨어 필요 여부 확인
2. 유선 네트워크로 펌웨어 패키지 설치
3. USB Wi-Fi 어댑터 사용 고려

#### 그래픽 드라이버 문제
**해결 방법:**
1. 먼저 오픈소스 드라이버 사용
2. 시스템 안정화 후 독점 드라이버 설치
3. 통합 그래픽을 대안으로 준비

#### 사운드 출력 불가
**해결 방법:**
1. ALSA 구성 확인
2. PulseAudio 설치
3. 커널 업데이트로 해결 가능
